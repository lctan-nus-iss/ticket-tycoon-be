# Ticket Tycoon

A multi-agent AI-powered investment simulation game backend built with Spring Boot 3.5 and Java 25. Players compete against AI opponents to reach $1,000,000 in net worth across quarterly market cycles. Every market event, AI decision, and piece of coaching is generated by a specialised LLM agent.

# Why
The aim of creating this financial game to educate people about investing, managing money, and building long-term wealth in an engaging and practical way. In the AI era, where automation is transforming industries and job stability can no longer be taken for granted, it is increasingly important for individuals to develop additional sources of income instead of relying solely on employment. Through realistic financial decisions and investment scenarios, the game aims to help players understand how to grow their assets, make informed choices, and achieve greater financial independence. Ultimately, I hope it will empower people to escape the rat race and build a more secure and fulfilling future.

## Agent Layer

Six specialised agents, each responsible for a distinct role in the game loop. All agents delegate LLM calls through the shared `LlmRouter`, which handles provider selection, fallback chains, and metric recording.

### EventAgent

**File:** `agent/EventAgent.java`

**Role:** Generates the quarterly market event that drives price movements for the entire round.

**How it works:**
- Receives the current quarter, year, macro context (rotates through 18 scenario templates), and a summary of previous events.
- Sends a structured prompt requesting a JSON `EventDTO` with: event name, description, affected assets with percentage change, optional bankruptcy risks, and narrative flavour.
- Applies geographic diversification constraints and prevents back-to-back severe events.
- On LLM failure, returns `EventDTO.fallback()` — a mild consolidation event.

**Primary provider:** DeepSeek  
**Fallback chain:** Anthropic → Ollama  
**Output:** `EventDTO` (JSON parsed from LLM response)

---

### AIPlayerAgent

**File:** `agent/AIPlayerAgent.java`

**Role:** Decides the trades each AI player makes every quarter, simulating distinct investor personalities.

**How it works:**
- Receives the AI player's archetype system prompt, current portfolio, available cash, current prices, the quarter's event summary, and win-progress context.
- Returns a list of `AIPlayerDecision.Trade` objects (`BUY asset amount` or `SELL asset percentage`).
- All AI players are executed **in parallel** using virtual threads.
- Validates every returned trade: caps BUY to 95% of available cash, clamps SELL to 1–100%, skips bankrupt assets, limits to 4 trades per turn.
- Falls back to a rule-based heuristic if the LLM call fails or JSON parsing errors.

**Primary provider:** Ollama (local)  
**Fallback chain:** DeepSeek → Anthropic  
**Output:** `List<AIPlayerDecision.Trade>`

---

### NarratorAgent

**File:** `agent/NarratorAgent.java`

**Role:** Generates immersive newspaper-style narrative excerpts that contextualise market events within the game's macro story.

**How it works:**
- Receives the event name, description, and the year/quarter.
- Returns a short prose passage written in a financial news style, grounding the mechanical event in storytelling.
- Output is displayed in the UI as a "news headline" each quarter.

**Primary provider:** DeepSeek  
**Fallback chain:** OpenAI → Anthropic  
**Output:** Plain text narrative string

---

### AdvisorAgent

**File:** `agent/AdvisorAgent.java`

**Role:** Provides personalised, educational portfolio analysis to the human player on demand.

**How it works:**
- Receives the player's full portfolio (positions, cash, net worth, trade history), current asset prices, and game context.
- Returns HTML-formatted analysis covering: portfolio composition, risk assessment, sector exposure, and actionable suggestions.
- Supports both **synchronous** (`POST /analyse/{playerId}`) and **streaming** (`GET /analyse/{playerId}/stream` — Server-Sent Events via `Flux<String>`) response modes.

**Primary provider:** Anthropic claude-sonnet-4-6  
**Fallback chain:** OpenAI  
**Output:** HTML string (or streaming chunks)

---

### CoachAgent

**File:** `agent/CoachAgent.java`

**Role:** Delivers concise 2–3 sentence teaching tips triggered by risky or suboptimal player behaviour.

**How it works:**
- A rule-based **pre-filter** evaluates the player's portfolio after each trade and determines whether coaching is warranted. Triggers:
  - `HIGH_CONCENTRATION_BANKRUPT_RISK` — single position >40% of portfolio in a bankruptcy-eligible asset
  - `EXCESSIVE_CONCENTRATION` — single position >50% of portfolio
  - `HIGH_VOLATILITY_LARGE_POSITION` — volatile asset (volatility >0.22) comprising >30% of portfolio
  - `CASH_DRAG` — holding >50% cash for more than 2 consecutive quarters
- Only calls the LLM when a trigger fires, keeping costs minimal.
- Returns `Optional<String>` — no message when no trigger matches.

**Primary provider:** Anthropic claude-haiku-4-5  
**Fallback chain:** DeepSeek → Ollama  
**Output:** `Optional<String>` coaching message

---

### DebriefAgent

**File:** `agent/DebriefAgent.java`

**Role:** Delivers a post-game retrospective analysis once the game ends, using extended reasoning.

**How it works:**
- Receives the full game history: all quarterly events, every player's trade log, final standings, and net worth progression.
- Uses DeepSeek R1 (a reasoning model) to produce a `DebriefResult` containing:
  - `analysis` — HTML-formatted retrospective covering key decisions, turning points, and lessons learned
  - `reasoningChain` — the model's chain-of-thought (extracted from `reasoning_content`) shown optionally in the UI
- The reasoning chain allows players to see *why* the AI drew each conclusion.

**Primary provider:** DeepSeek Reasoner (R1)  
**Fallback chain:** Anthropic  
**Output:** `DebriefResult` (HTML analysis + reasoning chain)

### AI Player Archetypes

**File:** `agent/AIPlayerArchetype.java`

Seven personality archetypes, each with a distinct investment philosophy encoded as a system prompt. The archetype determines which assets the AI player favours and how aggressively it trades.

| Archetype | Philosophy | Preferred Assets |
|---|---|---|
| `INDEXER` | Passive buy-and-hold diversification | ETFs (US, EU, Asia), bonds, gold |
| `SPECULATOR` | Aggressive momentum chasing, concentrated bets | Tech/biotech growth stocks, oil, tech ETFs |
| `INCOME_SEEKER` | Dividend yield maximisation | Dividend stocks, REITs, bonds |
| `GOLD_BUG` | Macro pessimist, crisis hedger | Gold, silver, bonds |
| `TECH_BULL` | AI and technology growth | Tech stocks, US tech ETF |
| `MACRO_TRADER` | Rotates aggressively on geopolitical events | Oil, gold, regional ETFs, defence |
| `ASIA_BULL` | China and Asia-Pacific growth focus | CN/Asia ETFs, silver, tech |

## LLM Integration Layer

### LlmRouter

**File:** `router/LlmRouter.java`

Central dispatcher implementing `LlmPort`. Every agent calls `llmRouter.complete(LlmRequest)` or `llmRouter.stream(LlmRequest)`.

**Responsibilities:**
1. Reads agent-specific configuration from `LlmProperties` (provider, model, temperature, max tokens, timeout, retries, fallback chain).
2. Builds the ordered provider chain: `[primary] + fallbacks`.
3. Iterates through the chain, attempting each provider until one succeeds.
4. Records Prometheus metrics for every attempt (success/failure, latency, token counts).

### Adapters

Five adapters, each implementing `LlmPort` and registered as a named Spring `@Component`:

| Component Name | Provider | Client | Notes |
|---|---|---|---|
| `anthropic` | Anthropic | Spring AI `AnthropicChatModel` | Uses `AnthropicChatOptions` per request |
| `openai` | OpenAI | Spring AI `OpenAiChatModel` | Uses `OpenAiChatOptions` per request |
| `deepseek` | DeepSeek | Spring AI `OpenAiChatModel` (qualified bean, custom base URL) | OpenAI-compatible API; forces temperature=1.0 for reasoner models |
| `gemini` | Google Gemini | Spring WebClient | Direct HTTP; Spring AI Gemini requires Vertex AI auth |
| `ollama` | Ollama (local) | Spring AI `OllamaChatModel` | Uses `OllamaOptions` with `numPredict` |

All adapters map the provider-agnostic `LlmRequest` to the provider's wire format and map the response back to `LlmResponse`.

## Game Flow

```
Human starts game
       │
       ▼
  ┌─────────┐
  │ Quarter │◄──────────────────────────────┐
  │  Start  │                               │
  └────┬────┘                               │
       │                                    │
       ▼                                    │
  EventAgent generates market event         │
       │                                    │
       ▼                                    │
  Prices updated (event effects + noise)    │
  Bankruptcy rolls applied                  │
       │                                    │
       ▼                                    │
  AIPlayerAgent runs for each AI player     │
  (parallel virtual threads)                │
       │                                    │
       ▼                                    │
  NarratorAgent generates news excerpt      │
       │                                    │
       ▼                                    │
  Dividends + cash interest distributed     │
       │                                    │
       ▼                                    │
  Win condition checked (≥ $1,000,000)      │
  ─── if no winner ───────────────────────►─┘
       │
       │ (winner found)
       ▼
  Game ends → DebriefAgent called

── During each quarter, human player may ──
  POST /buy or /sell
       │
       ├──► CoachAgent (if trigger fires)
       └──► AdvisorAgent (on demand, sync or streaming)
```

## REST API

Base path: `/api/game`

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/start` | Start a new game. Body: `{ humanName, aiArchetypeIds[] }` |
| `POST` | `/{gameId}/advance` | Advance to the next quarter (runs EventAgent + AIPlayerAgents + NarratorAgent) |
| `POST` | `/{gameId}/buy` | Human player buys an asset. Body: `TradeRequest` |
| `POST` | `/{gameId}/sell` | Human player sells an asset. Body: `TradeRequest` |
| `POST` | `/{gameId}/analyse/{playerId}` | Request synchronous portfolio analysis (AdvisorAgent) |
| `GET` | `/{gameId}/analyse/{playerId}/stream` | Request streaming portfolio analysis (SSE) |
| `POST` | `/{gameId}/debrief/{playerId}` | Request post-game debrief (DebriefAgent) |
| `GET` | `/archetypes` | List all available AI archetypes |
| `GET` | `/providers` | List all configured LLM providers |

## Observability & Metrics

### LlmObservabilityAspect

**File:** `aspect/LlmObservabilityAspect.java`

AOP aspect wrapping all LLM calls. On every completion it logs:
- Agent name, provider, model
- Input/output token counts
- Latency (ms)
- Estimated USD cost

**Cost table used for estimation:**

| Model | Input (per 1M tokens) | Output (per 1M tokens) |

| GPT-4o mini | $0.15 | $0.60 |
| Gemini 2.0 Flash | $0.075 | $0.30 |

## Configuration

Agent-level LLM configuration lives in `application.yml` under `llm.agents.*`:

```yaml
llm:
  agents:
    event:
      provider: deepseek
      model: deepseek-chat
      temperature: 0.8
      max-tokens: 1024
      timeout-seconds: 30
      retries: 2
      fallback-chain: [anthropic, ollama]

    ai-player:
      provider: ollama
      model: llama3.2
      temperature: 0.7
      max-tokens: 512
      timeout-seconds: 20
      retries: 1
      fallback-chain: [deepseek, anthropic]

    advisor:
      provider: anthropic
      model: claude-sonnet-4-6
      temperature: 0.6
      max-tokens: 2048
      timeout-seconds: 45
      retries: 2
      fallback-chain: [openai]

    coach:
      provider: anthropic
      model: claude-haiku-4-5-20251001
      temperature: 0.5
      max-tokens: 256
      timeout-seconds: 15
      retries: 1
      fallback-chain: [deepseek, ollama]

    debrief:
      provider: deepseek
      model: deepseek-reasoner
      temperature: 0.7
      max-tokens: 4096
      timeout-seconds: 120
      retries: 1
      fallback-chain: [anthropic]

    narrator:
      provider: deepseek
      model: deepseek-chat
      temperature: 0.9
      max-tokens: 512
      timeout-seconds: 20
      retries: 1
      fallback-chain: [openai, anthropic]
```

## Running Locally

**Prerequisites:** Java 25, Maven, Redis, PostgreSQL, Ollama (optional, for local LLM)

```bash
# Clone and build
mvn clean package -DskipTests

# Start dependencies
docker run -d -p 5432:5432 -e POSTGRES_DB=tickertycoon \
  -e POSTGRES_USER=$DB_USERNAME -e POSTGRES_PASSWORD=$DB_PASSWORD postgres:16
docker run -d -p 6379:6379 redis:7

# (Optional) Pull Ollama model for local AI player
ollama pull llama3.2

# Run
java -jar target/ticket-tycoon-be-*.jar
```

Server starts on `http://localhost:8080`.  
Frontend dev server expected at `http://localhost:5173` or `http://localhost:3000`.
